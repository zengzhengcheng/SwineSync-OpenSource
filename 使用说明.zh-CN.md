# SwineSync OpenSource 使用说明

## 1. 软件简介

软件名称：

- SwineSync OpenSource

当前开源功能包括：

- ECG 转换
- ECG 转换（使用缓存）
- 仅生成缓存
- 心率处理
- 文件夹检查
- 运动处理（旧版）
- 运动处理（新版）

说明：

- 开源版已删除“合成数据输出”功能
- 默认不输出合成数据

## 2. 目录结构

- [main.py](./main.py)：启动入口
- [gui.py](./gui.py)：主界面
- [cli_runner.py](./cli_runner.py)：后台任务执行器
- [config.json](./config.json)：配置文件
- [config_manager.py](./config_manager.py)：配置读写
- [translations.py](./translations.py)：中英文文案
- [dialogs.py](./dialogs.py)：文件确认对话框
- [neuronConvert.py](./neuronConvert.py)：ECG 转换调度
- [neuronDataClass.py](./neuronDataClass.py)：ECG 数据处理核心
- [heartClass.py](./heartClass.py)：心率处理
- [folder_inspector.py](./folder_inspector.py)：文件夹检查
- [moveClass.py](./moveClass.py)：旧版运动处理
- [moveClassNew.py](./moveClassNew.py)：新版运动处理
- [models](./models)：模型文件与推理代码
- [heartCorrect](./heartCorrect)：RR 校正逻辑

## 3. 安装与启动

安装依赖：

```bash
pip install -r requirements.txt
```

启动：

```bash
python main.py
```

## 4. 前后台运行机制

本版本已经做了前后台分离。

1. GUI 负责参数输入、文件选择、日志和状态显示。
2. 耗时处理放到独立 Python 子进程执行。
3. 日志会实时回传到前台。
4. 相比同进程执行，界面更不容易卡住。

## 5. 配置文件

配置文件：

- [config.json](./config.json)

主要字段：

- `language`
- `batch_size`
- `cache_size`
- `interpolation_algorithm`
- `correct`

`interpolation_algorithm` 可选值：

- `original_timestamps`
- `uniform_grid`

示例：

```json
{
  "language": "zh",
  "batch_size": 2,
  "cache_size": 1,
  "interpolation_algorithm": "original_timestamps",
  "correct": true
}
```

## 6. 中英文切换

点击界面中的语言切换按钮即可。

- 切换结果会自动保存到配置文件
- 下次启动会继续生效

## 7. 使用建议与硬件提醒

1. 第一个转换按钮相当于后两个流程合并。
2. 除非你有 64G 内存和 50 系以上显卡，否则更建议拆开跑。
3. 仅生成缓存时，可以通过增加 `cache_size` 提升并发。
4. 经验值是每 32G 空闲内存大约支持 6 个文件。
5. 转换 ECG（使用缓存）时，32G 内存、约 6 个文件，`cache_size=2` 通常够用。
6. 转换过程中如果某些页面心跳过少，程序会输出警告。

## 8. ECG 相关功能

### 8.1 转换 ECG

用途：

- 执行完整 ECG 转换流程

说明：

- 这个按钮相当于“先生成缓存，再基于缓存完成转换”
- 如果机器性能一般，更建议使用后两个按钮拆开处理

### 8.2 转换 ECG（使用缓存）

用途：

- 使用已经生成的缓存继续完成 ECG 转换

适用场景：

- 已经提前生成好缓存
- 希望降低一次性资源占用

### 8.3 仅生成缓存

用途：

- 只生成缓存，不做最终转换

使用建议：

- 增加 `cache_size` 可以提高缓存阶段并发
- 每 32G 空闲内存大约支持 6 个文件同时处理

文件识别规则：

- 文件名以 `Raw` 开头
- 或文件名中包含 `BMD`

## 9. 心率处理

用途：

- 汇总心率数据
- 导出 CSV
- 生成按天图片

操作流程：

1. 点击“心率处理”。
2. 选择心率数据所在文件夹。
3. 选择图片输出文件夹。
4. 选择结果 CSV 保存路径。
5. 等待后台处理完成。

输入建议：

- 心率原始文件建议为 CSV
- 文件名建议满足 `Raw*` 或包含 `BMD`

## 10. 文件夹检查

用途：

- 检查目录结构和文件命名是否合理

操作流程：

1. 点击“文件夹检查”。
2. 选择待检查的处理根目录。
3. 在日志窗口查看检查结果。

会重点检查：

- 日期文件夹是否建议以 `YYYYMMDD` 开头
- 文件名中的日期是否和目录日期一致
- 某个日期目录内文件是否过少
- 某个时间段里是否混入多头猪的数据标识

推荐目录组织：

- 处理文件夹 / 日期文件夹 / 具体传感器数据

推荐日期目录格式：

- `20250612`

注意：

- 一个时间段只建议放一头猪的数据

## 11. 运动处理

### 11.1 运动处理（旧版）

适用：

- 只有角速度的传感器

操作流程：

1. 点击“运动处理（旧版）”。
2. 选择待处理文件夹。
3. 选择图片输出文件夹。
4. 选择结果保存路径。

### 11.2 运动处理（新版）

适用：

- 包含加速度的传感器

操作流程：

1. 点击“运动处理（新版）”。
2. 选择待处理文件夹。
3. 选择图片输出文件夹。
4. 选择结果保存路径。

说明：

- 直接选择处理所在文件夹即可自动发现数据

## 12. 日志窗口

日志窗口会显示：

- 标准输出
- 标准错误
- 成功与失败提示

## 13. 常见问题

### 13.1 为什么没有合成数据输出功能

因为开源版已经删除该功能，并且默认不输出。

### 13.2 为什么界面没那么卡了

因为耗时处理已经改成独立后台子进程执行。
